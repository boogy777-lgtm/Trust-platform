# Historian And Database

## What truST Gives You Today

- observability outputs and historian-friendly runtime data:
  [Observability](../../operate/observability.md)
- protocol surfaces such as MQTT or OPC UA when your historian stack already
  consumes them:
  [MQTT](../external-systems/mqtt.md),
  [OPC UA](../external-systems/opc-ua.md)
- CLI/agent/export paths for custom pull/push flows:
  [Automate With CLI](../../start/automate-with-cli.md)

## Safest First Test

1. prove the value inside truST first
2. expose it through the chosen boundary
3. confirm the historian/database receives the same signal
4. only then widen the signal set
